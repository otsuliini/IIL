### Reporting Bugs 

#### Before Submitting a Bug Report

Make sure your bug report isn't making other people chase you for more info on the bug. I ask you to please investigate carefully, collect information, and describe the issue in a lot of detail. 
Please complete the following steps in advance to help me fix any potential bugs as fast as possible. 

- Make sure you are using the latest version.
- Determine that the bug is not on your side).
- Check if other users have solved/reported the same issue.
- Collect infromation about hte bug:
- Stack trace (Traceback)
- OS, Platform and Version (Windows, LInux, macOS, x86, ARM, etc.)
- Version of the interpreter, compiler SDK, runtime enviornment, package manager, etc.
- (optionally if it helps) your input and the output
- Can you reproduce the issue? And can you reproduce it with older versions?

#### How To Submit a Good Bug Report
> [!CAUTION]
> Please don't report security related issues, vulnerabilites or bugs incluWding sensitive infromation to the issue tracker, or anywhere in public. Instead sensitive bugs should be sent by email to: otso.reijonen@icloud.com. (thx)

This project uses GitHub issues to track bugs and errors. If you run into an issue just: 
- Open an [Issue](/issues/new).
- Explain the behavior you would expect and the actual behavior.
- Please provide as much context as possible and describe the *reproduction steps* that someone else should follow to reproduce the issue.
- Provide the info you collected in the privous section.

Once the issue is filed: 
- We'll hope that someone fixes it...

> [!TIP]
> All in all, I don't take this stuff seriously lol. So just don't be an idiot and try your best. 

### Suggesting Enhancements

So you have an idea for a feauture to add to IIL? This section guides you through submitting an enhance suggestion.  

#### Before Submitting an Enhancement

- Make sure that you are using the latest version.
- Read the [documentation](docs/)
- Perform a [search}(/issues) to see if the enhancement has already been suggested. If it has just add a comment or something to the existing issue instead of opening a new one.
- Find out whether your idea fits with the scope and aims of IIL.

#### How To Submit a Good Enhancement Suggestion?

Enhancement suggestions are tracked as [GitHub issues](/issues). 

- Use a **Clear and descriptive** title for the issue.
- Provide  a **step-by-step description of the suggested enhancement** in as many details as you believe is neccessary.
- (Optional) describe the current behavior and explain which behavior you expected to see and why.
- **Explain why this enhancement would be useful** to IIL users. You may also want to point out other projects that have done something better/projects that can be used as inspiration.

### Commit messages
Make sure your commit messages are concise and clear.
